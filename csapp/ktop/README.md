# ktop version 2.0

## 发展

ktop 在最开始开发的初衷主要就是为了了解`Electron`和`Wpf`两种桌面程序开发有什么样的区别，
在使用electron开发了第一个版本的ktop后，紧接着使用c#开发了同样功能的ktop，
通过这两个实现方式的对比，发现Electron虽然有些笨重，但是在界面开发效率上是Wpf无法比拟的，
当然这可能也是因为我更擅长前端开发。Wpf虽然界面开发上要比Web更加复杂繁琐一些，
但是其最终包大小和性能表现让我惊讶，因此也在wpf版本上对ktop做了更多的开发工作。

基于chrome内核的electron应用应该可以在任何环境下运行，但是wpf要求用户安装.Net环境，
内部使用了很多Windows API，基本上限制死了软件受众，不过这并没有关系，
因为ktop本来就是为win用户准备的小工具。

## ktop是什么

ktop v2 准备开发以下功能特性：

- [x] 监控系统的性能，包括cpu和内存
- [ ] 快速执行命令
- [ ] Crontab周期执行任务，包括使用Crontab执行外部脚本
- [ ] 网址管理与快捷打开
- [ ] 应用程序管理与快捷打开
- [ ] 提供基于xray-core、sing-box的系统代理工具

由于该工具只有我一人维护，因此上面的规划可能不会全部实现。

## 安装使用

ktop并不提供任何安装程序 installer.exe，在发布页面只会发布 软件的程序压缩吧，
解压后运行目录内的`ktop.exe`即可。

在每次运行之前，你可以搜索并编辑目录下的
`appConfig.json`文件，这是ktop的主要配置文件，
在这里可以事先规定一些行为，比如启动后是否尝试在用户桌面创建软件快捷方式等等。

## 卸载

删除ktop主程序所在目录即可，同样的你也可以随意移动这个目录。

## 插件

ktop的插件功能被划分在“命令”功能区，因为在ktop里，
插件依然是通过cmd命令调用exe或其他可执行程序来完成。

在ktop里，插件实际上就是一些免费开源的程序，
ktop在发布时会内置一些插件，比如截图工具。

您也可以使用ktop管理您电脑上的任何可执行程序，
就像桌面图标那样轻松使用。
